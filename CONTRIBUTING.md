# Contributing to ssc_assignment_1  

Thanks for your interest in contributing! By participating, you agree to follow our guidelines.

## How to Contribute

1. **Bug reports**
   - Check if the issue already exists before opening a new one.
   - Include clear steps to reproduce the problem.
   - Provide environment details (OS, version, dependencies).

2. **Feature requests**
   - Describe the motivation and expected behavior.
   - Explain why the feature is needed.
   - Reference similar features or use cases if possible.

3. **Pull Requests**
   - Clone the repository and create a feature branch.
   - Follow the code style and formatting used in the project.
   - Write tests for new functionality if applicable.
   - Ensure all tests pass before submitting.
   - Provide a clear description of what your PR does.

4. **Communication**
   - Be respectful and constructive.
   - Do unto others as you would have them do unto you.  

## Getting Help

- Check the README and existing issues for guidance.
- Ask questions on the issue tracker before starting a major change.

Thank you for helping make ssc_assignment_1 better!
